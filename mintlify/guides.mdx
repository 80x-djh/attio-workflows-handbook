Guides are complete, follow-along Attio workflow builds, each with the exact trigger, blocks, credit math, and a test plan you can run. Every build names its trigger and configures every block with real field values. It shows the per-run credit cost, then ends with a test you fire yourself. Ship it into your own workspace and watch it run.

Each build stands alone, so grab the one you need. If you would rather learn a whole job in order, the guided paths in [Learn](/learn) walk you through these same builds with the mental model first. These guides assume you have finished [Build your first workflow](/learn/build-your-first-workflow). When a concept is unclear, [How the engine works](/explanation) covers the why.

## What makes a good build

The best Attio workflow is boring. It starts with a precise trigger and an early filter that is free to run. From there it does the least it can. That means one small data lookup, one clear write or external action, then a check in the run viewer. It also names an owner and a way to roll it back. Reach for agents, the [Send HTTP request](/reference/utilities/send-http-request) block, and the [Execute code](/reference/calculations/execute-code) block when they remove real complexity, not because they are available.
