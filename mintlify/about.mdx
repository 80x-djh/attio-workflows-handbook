The **Attio Workflows Handbook** is a community-maintained, source-backed guide for people building go-to-market systems in Attio. It is written by [Dan Hull](https://80x.ai), who builds Attio workspaces for venture capital (VC) funds and startups full-time. The handbook is not affiliated with or endorsed by Attio Ltd. For how the product behaves, the official Attio Help Center is still the source of truth.

What the handbook adds on top is implementation judgment: how to actually build these systems well. You get patterns, step-by-step recipes, migration checklists, cost tradeoffs, and go-to-market (GTM) examples.

## Contribution principles

Good contributions raise the handbook's accuracy and stay inside its scope. Follow these principles when you add or edit content.

- Prefer tested workspace behavior over speculation.
- Link to the relevant Attio Help Center page when you document product behavior.
- Keep examples GTM-specific: pipeline, routing, enrichment, outbound, product signals, fund workflows, and customer operations.
- Mark assumptions clearly.
- Never publish private customer data, screenshots, or workspace-specific IDs.
- Include the cost model and a safety guard whenever a workflow can burn credits at scale.

## Good first contributions

These changes are small, high-value, and a good place to start.

- Add a missing block gotcha.
- Add a recipe from a real GTM workflow.
- Improve a migration note for a legacy block.
- Add a troubleshooting error and its exact fix.
- Update a page when Attio changes a block's behavior.

## Verification standard

Each [block reference](/reference) page aims to carry the same six elements. That way you can trust a page at a glance.

- The block type and credit cost.
- The inputs and outputs.
- When to use it.
- Common gotchas.
- One GTM example.
- The official source link.

The goal is not to out-document Attio's official reference. The goal is to make builders faster and safer.
