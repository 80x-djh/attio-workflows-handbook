The **Send HTTP request** block (Utilities) calls an external URL from a workflow and returns the response. It is how a workflow reaches enrichment providers, internal services, or any system with a REST API, and it is the only block that sends an arbitrary outbound request. Treat every call as unreliable: the block returns a success flag you must check before you write the returned data back into Attio.

| --- | --- |
| **Category** | Utilities |
| **Credit cost** | 1 credit |
| **Engine** | New Workflows engine (June 2026+) |
| **Plan availability** | All plans that include workflows |
| **Limits** | 2-minute request timeout. |

## Inputs

| Input | Type | Required | Default | Notes |
| --- | --- | --- | --- | --- |
| Method | Enum | Yes | — | One of `DELETE`, `GET`, `HEAD`, `PATCH`, `POST`, or `PUT`. |
| URL | Text | Yes | — | Static text, [variables](/reference/variable-syntax), or both. |
| Headers | Key-value pairs | No | None | Authentication and content headers, subject to platform restrictions. |
| Content-Type | Text | Conditional | None | Required when the request sends a body. |
| Body | Text | No | None | The request payload, usually JSON. |

## Outputs

| Output | Type | Example | Notes |
| --- | --- | --- | --- |
| Status code | Number | `200` | The HTTP status code the endpoint returned. |
| Response body | Text | `{"name":"Acme","employees":240}` | The raw response body, returned as text. |
| Success | Boolean | `true` | Whether the response was successful. |

Reference the response body downstream as `{{ steps.send_http_request.response_body }}`, the status code as `{{ steps.send_http_request.status_code }}`, and the success flag as `{{ steps.send_http_request.success }}`.

## Behavior

The block sends a single request and waits for the response before the workflow continues. The request times out after 2 minutes; an endpoint that does not respond within that window fails the block rather than hanging the run.

The response body is returned as text. To use individual fields downstream, follow the block with [Parse JSON](/reference/utilities/parse-json), which converts the string into typed [variables](/explanation/variables). The block does not parse the response for you.

The success flag reflects the transport-level outcome, so check it before you act on the body. An endpoint can return a `200` status with an error payload, or a `4xx` status with a body that explains the problem; gate any record write on the success flag and, where the API uses status codes meaningfully, on the status code as well.

## Gotchas

- **The response arrives as raw text, not fields.** The block returns the body as a string, so `{{ steps.send_http_request.response_body }}` cannot be mapped field-by-field on its own. Add a [Parse JSON](/reference/utilities/parse-json) block after it and reference the parsed outputs downstream.
- **A request that never returns fails at 2 minutes.** A slow or unresponsive endpoint holds the run open until the 2-minute timeout, then fails the block. Point the URL at endpoints that respond quickly, and branch on the success flag so a timeout does not silently skip later writes.
- **Writing the body without checking success corrupts records.** A failed call can still carry a body, and writing it back overwrites good data with an error message. Add an [If](/reference/conditions/if) block on `{{ steps.send_http_request.success }}` and only write when it is `true`.
- **A missing Content-Type drops the body.** Requests that send a payload need a Content-Type header; without it the endpoint may ignore or reject the body. Set `Content-Type` to `application/json` whenever the Body field is populated.

## Examples

### Enriching a new company from an external API

This workflow calls an enrichment provider when a company gains a domain, then parses the response and updates the record. The example sends a `GET` request with the domain in the path and an API key in the header.

| Field | Value |
| --- | --- |
| Method | `GET` |
| URL | `https://api.enrich.example/v1/companies/{{ trigger.record.domain }}` |
| Headers | `Authorization: Bearer {{ secrets.enrich_api_key }}` |
| Content-Type | (none) |
| Body | (none) |

The block returns `200` with a JSON body such as `{"name":"Acme","employees":240}`. A following [Parse JSON](/reference/utilities/parse-json) block extracts `employees`, and an [Update record](/reference/records/update-record) block writes it to the company.

### Posting a deal to an external billing system

This workflow notifies a billing service when a deal reaches closed-won. The example sends a `POST` request with a JSON body built from deal variables.

```json
{
  "method": "POST",
  "url": "https://api.billing.example/v1/accounts",
  "headers": {
    "Authorization": "Bearer {{ secrets.billing_api_key }}",
    "Content-Type": "application/json"
  },
  "body": {
    "account_name": "{{ trigger.record.name }}",
    "owner_email": "jane@acme.com",
    "amount": "{{ trigger.record.amount }}"
  }
}
```

The block returns `201` with the new account's ID in the body, and `{{ steps.send_http_request.success }}` is `true`. An [If](/reference/conditions/if) block confirms success before an [Update record](/reference/records/update-record) block stores the returned ID on the deal.

### Handling a failed enrichment call

This example shows the edge case: the enrichment endpoint is down and the request times out. The configuration matches the first example, but the endpoint does not respond within 2 minutes.

| Field | Value |
| --- | --- |
| Method | `GET` |
| URL | `https://api.enrich.example/v1/companies/acme.com` |
| Headers | `Authorization: Bearer {{ secrets.enrich_api_key }}` |

The block fails at the 2-minute timeout and `{{ steps.send_http_request.success }}` is `false`. The downstream [If](/reference/conditions/if) branch routes to a [Create task](/reference/tasks/create-task) block for a manual follow-up instead of writing an empty enrichment result over the company.

## Credit cost worked example

A single **Send HTTP request** costs 1 credit per run. An outbound enrichment workflow that fires on each of 500 new companies per month spends 500 credits on this block alone. A following [Parse JSON](/reference/utilities/parse-json) block is free, so it adds nothing; an [Update record](/reference/records/update-record) block adds 1 credit per run, taking the path to 2 credits and 1,000 credits per month. Check that monthly total against your plan's allowance on [Credits & pricing explained](/reference/credits-and-pricing).
