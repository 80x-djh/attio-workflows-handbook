Troubleshooting a failing Attio workflow starts in the run viewer, not in the block editor. The run viewer is the screen that shows each run step by step. For every block, it shows the inputs the block received and the outputs it produced. Read it from the top down to find where the first wrong value entered the chain. Open it before you change any configuration. Most "broken" workflows are wired correctly and simply fed bad data. For the run viewer, the difference between draft and published runs, and how retries work, see [Runs, debugging & publishing](/explanation/runs-and-debugging).
