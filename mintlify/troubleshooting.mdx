Troubleshooting a failing Attio workflow starts in the run viewer, not in the block editor. The run viewer shows you the exact inputs each block received and the outputs it produced, so you can trace upstream to where the first wrong value entered the chain. Reach for it before you change any configuration, because most "broken" workflows are wired correctly and fed bad data. For the run viewer, draft-versus-published runs, and retry semantics, see [Runs, debugging & publishing](/explanation/runs-and-debugging).
