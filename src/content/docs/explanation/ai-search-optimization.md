This handbook is built for humans first, but it should also be easy for AI assistants to read.

## What is already in place

- Clean Starlight documentation routes.
- Descriptive page titles and descriptions.
- One topic per URL.
- Source links back to official Attio docs.
- `/llms.txt` generated by `starlight-llms-txt`.
- Tables for decision points, costs, and block inputs.
- Recipe pages with trigger, build, credit model, and safety guard sections.

## Authoring rules

- Answer the exact question in the first screen.
- Use stable names: "Attribute value changed", not "the changed field trigger".
- Include "when not to use this" when a block is risky.
- Put cost and failure behavior near the top.
- Use internal links to connect concepts, references, and recipes.
- Avoid vague claims like "powerful" unless followed by a concrete use case.

## Maintenance rule

Every page that describes product behavior should include an official source link or a "verified in workspace" note. AI-optimized does not mean confident-sounding. It means structured, source-backed, and easy to update.
